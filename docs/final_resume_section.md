# Final Resume Section

## Project Title

Broadband Customer 360 & Churn Analytics — Snowflake + Databricks

## Resume Project Description

Built an end-to-end broadband analytics platform in Snowflake and Databricks to track monthly Active, New, and Churned customers, calculate revenue impact, create Customer 360 analytics, identify churn-risk customers, and monitor data quality through Power BI dashboards.

## Resume Bullets

- Designed and built a dual-platform Broadband Customer 360 and Churn Analytics pipeline using Snowflake, Databricks, SQL, Python, and Power BI.
- Implemented Bronze, Silver, and Gold data layers for raw ingestion, cleaned data, business-ready reporting, and analytics consumption.
- Built monthly Active/New/Churn reporting with churn rate, net adds, new revenue, churn revenue, and net revenue impact.
- Developed a Customer 360 monthly Gold table combining customer profile, subscription, payment, network usage, trouble ticket, and on-top product data.
- Created churn-risk features using rolling 3-month late payment count, trouble ticket count, usage drop, speed drop, service month, and product adoption.
- Added production-style data quality monitoring with row count, duplicate key, null key, freshness, and PASS/WARNING/FAIL checks.
- Built quarantine handling to capture invalid Bronze records with rule name, severity, business key, error message, and original record payload.
- Created Snowflake vs Databricks reconciliation checks to validate consistent business outputs across warehouse and lakehouse implementations.
- Added regression tests to validate data completeness, uniqueness, KPI formulas, churn-risk values, and DQ monitoring status.
- Built a Power BI dashboard with Executive Summary, Customer Behavior, Churn Risk, Data Quality, and Platform Story pages.
